Java基础之面向对象

Posted on 2021-01-09 In java基础

面向对象的三个基本特征是：封装、继承、多态。

方法重写与方法重载

面向对象之封装

用大白话来讲，封装就是将自己类内部的方法、变量设为私有、不允许外部类直接访问。要想访问、就必须通过我指定的get() set()方法来访问。封装和private直接相关。

封装的概念

  • 1.封装概述

    • 是面向对象编程语言对客观世界的模拟，客观世界里成员变量都是隐藏在对象内部的，外界是无法直接操作的
    1. 封装原则
    • 将类的某些信息隐藏在类内部，不允许外部程序直接访问，而是通过该类提供的方法来实现对隐藏信息的操作和访问
    • 成员变量private，提供对应的getXxx()/setXxx()方法
    1. 封装好处
    • 通过方法来控制成员变量的操作，提高了代码的安全性
    • 把代码用方法进行封装，提高了代码的复用性

private关键字

概念：

  • private是一个修饰符，可以用来修饰成员（成员变量，成员方法）
  • 被private修饰的成员，只能在本类进行访问
  • 针对private修饰的成员变量，如果需要被其他类使用，提供相应的操作
    • 提供“get变量名()”方法，用于获取成员变量的值，方法用public修饰
    • 提供“set变量名(参数)”方法，用于设置成员变量的值，方法用public修饰

面向对象之继承

用大白话来讲就是，将多个对象共同的部分抽象称为一个父类，这些共同的属性就直接在父类中定义就可以了，子类直接继承父类，代码更简洁、修改时只需修改一次，很实用。比如猫、老虎、豹子都是属于猫科动物，我就直接定义一个猫科动物的父类，这三个家伙直接继承之。

继承的概念

  • 定义
    • 继承是面向对象三大特征之一，可以使得子类具有父类的属性和方法，还可以在子类中重新定义，以及追加属性和方法
  • 格式
    • 格式：class 子类 extends 父类 { }
    • 举例：class Dog extends Animal { }
  • 子父类的关系
    • 继承可以让类与类之间产生关系，子父类关系，
    • 子类则可以使用父类中非私有的成员
    • 子类也可以有自己特有的内容

继承的好处和弊端

  • 好处
    • 提高了代码的复用性(多个类相同的成员可以放到同一个类中)
    • 提高了代码的维护性(如果方法的代码需要修改，修改一处即可)
  • 弊端
    • 继承让类与类之间产生了关系，类的耦合性增强了，当父类发生变化时子类实现也不得不跟着变化，削弱了子类的独立性
  • 应用场景
    • 使用继承，需要考虑类与类之间是否存在is..a的关系，不能盲目使用继承
    • 如：苹果和水果 猫和动物

面向对象之多态

一个对象的多种状态？基于封装和继承。多态指在编译层面无法确定最终调用的方法体，在运行期由 JVM 动态绑定，调用合适的重写方法。多态一般专指重写。

多态的概念

  • 定义

    • 同一个对象，在不同时刻表现出来的不同状态

  • 前提（同时满足）

    • 要有继承或者实现关系、

    • 要有方法的重写（子类中重写父类的方法）

    • 有父类引用指向子类对象（在测试类中实现）

多态的特点和转型

  • 好处和弊端

    • 好处、
      • 提高程序的扩展性。定义方法时候，使用父类型作为参数，在使用的时候，使用具体的子类型参与
    • 操作弊端
      • 不能使用子类的特有成员

  • 多态的转型

    • 向上 转型

      父类引用指向子类对象

    • 向下转型

      父类引用转为子类对象

  • 多态成员的访问特点

    • 成员变量
      • 编译看父类，运行看父类
    • 成员方法
      • 编译看父类，运行看子类
    • 原因
      • 成员方法在子类中存在重写 ，而成员变量没有

        方法重写与方法重载

方法重写

方法重写：子类对父类的允许访问的方法的实现过程进行重新编写，返回值的形参都不能改变。即外壳不变，核心重写。

特点：对编译器来说，方法名称和参数列表组成了一个唯一键，称为方法签名，JVM 通过
方法签名决定调用哪种重载方法。重载在编译时可以根据规则知道调用哪种目标方法，因此属于静态绑定。

方法重载

方法重载：在一个类里面，方法名相同，而参数不同，返回值可同可不同。每个重载的方法（或构造函数）都必须有一个独一无二的参数类型列表。

特点：重写方法访问权限不能变小，返回类型和抛出的异常类型不能变大，必须加
@Override 。
